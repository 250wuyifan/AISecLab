DVMCP 实战靶场

Damn Vulnerable MCP Server - 包含 10 个难度递增的 MCP 安全挑战， 涵盖提示注入、工具投毒、权限过度、令牌窃取、代码执行等多种攻击向量。

快速开始 点击展开
1
启动 DVMCP 服务（Docker 一键启动，推荐）
使用 Docker 一次性启动全部 10 个挑战服务（端口 9001-9010）：
# 克隆 DVMCP 项目（如果还没有） git clone https://github.com/250wuyifan/damn-vulnerable-MCP-server-CN.git # 构建并启动（端口 9001-9010） cd damn-vulnerable-MCP-server-CN docker build -t dvmcp . docker run -d --name dvmcp -p 9001-9010:9001-9010 dvmcp
不用 Docker？手动启动方式
# 进入 DVMCP 项目目录 cd damn-vulnerable-MCP-server-CN pip install -r requirements.txt # 一键启动全部 10 个挑战 bash start_sse_servers.sh # 或启动单个挑战（如挑战 1，端口 9001） python challenges/easy/challenge1/server_sse.py
2
启动本地 LLM（Ollama）
ollama serve # 启动 Ollama 服务 ollama pull qwen2.5:7b # 拉取推荐模型
3
检查状态并开始挑战
MCP 服务 {{ running_count }}/{{ total_challenges }} 运行
完成 {{ completed_count }}/{{ total_challenges }}
状态正常后，点击下方的挑战卡片进入详情页。页面内置聊天界面，可直接与 LLM + MCP 服务器交互。
使用其他 MCP 客户端（Cursor / Claude Desktop / Cline）

除了页面内置的交互方式，你也可以使用其他支持 MCP 协议的客户端连接 DVMCP 服务器：

编辑 ~/.cursor/mcp.json：

{ "mcpServers": { "dvmcp-挑战1": { "url": "http://localhost:9001/sse" }, "dvmcp-挑战2": { "url": "http://localhost:9002/sse" } } }

编辑配置文件（macOS: ~/Library/Application Support/Claude/claude_desktop_config.json）：

{ "mcpServers": { "dvmcp": { "command": "npx", "args": ["mcp-remote", "http://localhost:9001/sse"] } } }

在 VS Code 中安装 Cline 插件后，在 MCP 设置中添加：

{ "mcpServers": { "dvmcp": { "command": "npx", "args": ["mcp-remote", "http://localhost:9001/sse"] } } }
