MCP 组件与主要风险（字节安全）
MCP 组件 主要风险
MCP Servers同通用 Tool 攻击面：N-day、鉴权问题、RCE/SSRF
MCP Server 市场供应链攻击、越权、凭证泄露；rug pull、tool poisoning
MCP Clients添加/安装 Server 时命令执行、SSRF（盲信 URL）
MCP Hosts恶意 Server 返回内容 → 间接 Prompt Injection → 窃取聊天、跨 Server Tool 调用
整体一旦信任恶意/被污染的 Server，Agent 全盘沦陷；需沙箱 + 信任管理 + 输出防护
修复建议
  • 沙箱隔离：MCP Server 调用放 Docker/VM/无网络沙箱。
  • 信任管理：审核源码、签名验证、白名单；不盲信市场 Server。
  • 输出校验：Host 收到 Server 返回前过滤/扫描 Prompt Injection。
  • 最小权限：OAuth scope 收紧；高危操作二次确认。
