DVMCP 实战靶场

Damn Vulnerable MCP Server - 包含 10 个难度递增的 MCP 安全挑战， 涵盖提示注入、工具投毒、权限过度、令牌窃取、代码执行等多种攻击向量。

快速开始

1
启动 DVMCP 服务
在终端中进入 DVMCP 目录并启动服务：
cd {{ dvmcp_path }} # 启动所有挑战（端口 9001-9010） python challenges/easy/challenge1/server_sse.py & python challenges/easy/challenge2/server_sse.py & # ... 或启动单个挑战
启动单个挑战： python challenges/easy/challenge1/server_sse.py（端口 9001）
2
确保本地 LLM 运行
靶场使用本地大模型（Ollama）进行交互，请确保已启动：
# 启动 Ollama 服务 ollama serve # 拉取推荐模型（如果还没有） ollama pull qwen2.5:7b
3
检查服务状态
MCP 服务 {{ running_count }}/{{ total_challenges }} 运行
完成进度 {{ completed_count }}/{{ total_challenges }}
4
进入挑战，直接在页面上操作
点击下方的挑战卡片进入详情页，页面内置了聊天交互界面， 你可以直接与本地 LLM + MCP 服务器交互，无需配置任何外部客户端！
页面内聊天
自动调用工具
本地 LLM
使用其他 MCP 客户端（Cursor / Claude Desktop / Cline）

除了页面内置的交互方式，你也可以使用其他支持 MCP 协议的客户端连接 DVMCP 服务器：

编辑 ~/.cursor/mcp.json：

{ "mcpServers": { "dvmcp-挑战1": { "url": "http://localhost:9001/sse" }, "dvmcp-挑战2": { "url": "http://localhost:9002/sse" } } }

编辑配置文件（macOS: ~/Library/Application Support/Claude/claude_desktop_config.json）：

{ "mcpServers": { "dvmcp": { "command": "npx", "args": ["mcp-remote", "http://localhost:9001/sse"] } } }

在 VS Code 中安装 Cline 插件后，在 MCP 设置中添加：

{ "mcpServers": { "dvmcp": { "command": "npx", "args": ["mcp-remote", "http://localhost:9001/sse"] } } }
